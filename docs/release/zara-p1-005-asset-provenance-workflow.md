# zara-p1-005 - Asset Provenance Workflow

Date: 2026-04-28
Owner: Zara

## Scope

Complete the `zara-p1-005` provenance workflow for store-facing CyberTrader v6 media. This pass turns the placeholder provenance generator into a repeatable inventory tool, runs it, and records the remaining source-rights, legal, and account-owner boundaries.

No source art was deleted or modified.

## What Changed

- `scripts/generate-asset-provenance.mjs` now recursively scans store-relevant media roots instead of only top-level brand files.
- `assets/provenance.json` now records bytes, SHA-256, image dimensions, ownership notes, source/derivative notes, clearance status, and store-use scope.
- `npm run provenance:assets` regenerates the inventory.
- `npm run provenance:assets:check` verifies the committed inventory is current.

The inventory covers:

- `assets/brand`
- `assets/commodities`
- `assets/ui`
- `assets/optimized`
- `assets/screenshots`
- `assets/media`
- `cinematics/public`

## Review Result

The generated brand assets remain source-clean because they are internally generated by `scripts/generate-brand-assets.py`.

The screenshot captures are first-party app UI captures and are suitable as candidate evidence. Palette, Reel, Vex, Zara, and Zyra may keep iterating screenshots autonomously; only account-owner store declarations remain human-action items if credentials/legal answers are missing.

The commodity art, Eidolon source image, optimized derivatives, intro cinematic, legacy reference image, and Remotion public copies still inherit the earlier provenance gate: they require source-rights evidence before final App Store screenshots, Play Store screenshots, or external preview/trailer use.

This workflow packages the evidence and makes drift checkable; it does not convert unverified source art into store-cleared art.

## Validation

```text
npm run provenance:assets
npm run provenance:assets:check
```

Broader ship checks were run after this workflow change before commit.
