# zara-p1-005 - Asset Provenance Workflow

Date: 2026-04-28
Owner: Zara

## Scope

Complete the `zara-p1-005` store-media provenance workflow for CyberTrader v6. This pass replaces the placeholder brand-only provenance generator with a repeatable inventory tool for the v6 media that still matters to store screenshots, preview staging, and trailer planning.

No Dev Lab 3D office assets are included. No source art was deleted or modified.

## What Changed

- `scripts/generate-asset-provenance.mjs` now recursively scans store-relevant media roots.
- `assets/provenance.json` now records bytes, SHA-256, image dimensions, ownership notes, source or derivative notes, clearance status, and store-use scope.
- `npm run provenance:assets` regenerates the inventory.
- `npm run provenance:assets:check` verifies the committed inventory is current.

The inventory covers:

- `assets/brand`
- `assets/commodities`
- `assets/ui`
- `assets/optimized`
- `assets/screenshots`
- `assets/media`
- `cinematics/public`

## Review Result

The generated brand assets remain source-clean because they are internally generated by `scripts/generate-brand-assets.py`.

The screenshot captures are first-party app UI captures and are suitable as candidate evidence, but they still require Palette/Zoro final creative approval before store submission.

The commodity art, Eidolon source image, optimized derivatives, intro cinematic, legacy reference image, and Remotion public copies still inherit the earlier provenance gate: they require Palette/Zoro source attestation before final App Store screenshots, Play Store screenshots, or external preview/trailer use.

This workflow packages the evidence and makes drift checkable; it does not convert unverified source art into store-cleared art.

## Validation

```text
npm run provenance:assets
npm run provenance:assets:check
```

Broader ship checks were run after this workflow change before commit.
