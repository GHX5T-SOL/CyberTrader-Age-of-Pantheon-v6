# palette-p1-003 Screenshot-safe Visual State Presets

## Purpose
Create a set of visual state presets for the app that are safe for inclusion in App Store screenshots and preview video. These presets capture the UI in a clean, data‑neutral state that meets Apple and Google screenshot guidelines (no sensitive data, no placeholder text, deterministic visuals).

## Preset Definitions
| Route | State | Description | Asset Key |
|-------|-------|-------------|-----------|
| `/home` | **Idle** | Home screen showing empty inventory, no active missions, Energy/Heat at baseline, and a default market price view. | `screenshot-home-idle` |
| `/terminal` | **Ready** | Terminal with a single buy order input populated, no trade executed, and telemetry chips showing 0 BOL, Energy 100, Heat 0. | `screenshot-terminal-ready` |
| `/market` | **Market Overview** | Market screen with the first commodity (VBLM) highlighted, price chart displayed, no active orders. | `screenshot-market-overview` |
| `/missions` | **Mission List** | Missions screen showing the default tutorial mission highlighted, no progress bar filled. | `screenshot-missions-list` |
| `/profile` | **Profile Overview** | Profile screen with placeholder avatar, rank 1, and zero XP displayed. | `screenshot-profile-overview` |

## Asset Generation Process
1. Run the app in the Expo web export (`npm run export:web`).
2. Navigate to each route listed above and ensure the deterministic state is achieved (e.g., clear any persisted storage before capture).
3. Use a headless Chromium screenshot utility (e.g., `playwright` or `puppeteer`) to capture a 1242 × 2688px PNG for each route.
4. Store the generated images under `assets/screenshots/` with the filenames matching the **Asset Key**.
5. Verify that each image meets Apple/Google size and aspect‑ratio requirements and contains no personal or placeholder data.

## Automation Stub (to be added to `scripts/`)

- Images are referenced in `app.json` under the `ios.splash` and `android.splash` sections for use in the store.
