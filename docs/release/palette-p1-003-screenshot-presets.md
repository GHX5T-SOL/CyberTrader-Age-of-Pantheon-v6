# palette-p1-003 Screenshot-safe Visual State Presets

## Purpose
Create a set of visual state presets for the app that are safe for inclusion in App Store screenshots and preview video. These presets capture the UI in a clean, data‑neutral state that meets Apple and Google screenshot guidelines (no sensitive data, no placeholder text, deterministic visuals).

## Preset Definitions
| Route | State | Description | Asset Key |
|-------|-------|-------------|-----------|
| `/home` | **Idle** | Home screen after local tutorial completion with market list, first-session objective, and no personal data. | `screenshot-home-idle` |
| `/terminal` | **Ready** | Terminal with the starter VBLM buy order staged and local demo telemetry visible. | `screenshot-terminal-ready` |
| `/market` | **Market Overview** | Market route resolves to the terminal market list with no trade executed. | `screenshot-market-overview` |
| `/missions` | **Mission List** | Missions screen showing contact and archive state without user-specific data. | `screenshot-missions-list` |
| `/menu/inventory` | **Inventory Overview** | Inventory screen showing the current storage state without personal data. | `screenshot-inventory-overview` |
| `/menu/profile` | **Profile Overview** | Profile screen with local demo identity, rank 1, and no wallet address. | `screenshot-profile-overview` |

## Asset Generation Process
1. Run `npm run capture:screenshots`.
2. The package script builds the Expo web export, serves `dist/`, starts a local demo session, completes tutorial onboarding, fixes the browser clock at 2026-04-28 10:00 UTC, and captures the preset routes with Playwright Chromium at a 414 x 896 mobile viewport with 3x device scale, producing 1242 x 2688 PNGs.
3. The script writes PNG captures to `assets/screenshots/` and rejects tiny placeholder files or wrong dimensions.
4. Verify that each image meets Apple/Google aspect-ratio requirements and contains no personal, placeholder, or policy-unsafe data before using it in store listings.

## Automation
`scripts/capture-screenshot-presets.mjs` is now an executable capture utility. It does not generate 1x1 placeholder images; it fails if the web export is missing, a route marker is missing, a PNG is too small, dimensions are wrong, or browser console/page errors appear during capture. `scripts/generate-screenshot-presets.mjs` is retained as a compatibility wrapper.
